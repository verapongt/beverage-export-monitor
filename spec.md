# Spec: Dashboard ติดตามการส่งออกหมวดเครื่องดื่มใกล้เคียง Sappe / Mogu Mogu

## เป้าหมาย

สร้าง Web App แบบ Custom Dashboard สำหรับติดตามภาพรวมการส่งออกไปต่างประเทศของหมวดเครื่องดื่มที่ใกล้เคียงกับสินค้า Sappe / Mogu Mogu เพื่อให้นักลงทุนและนักวิเคราะห์หุ้นใช้ดูแนวโน้มอุตสาหกรรม ประเมินการเติบโตของตลาดต่างประเทศ และใช้เป็นข้อมูลประกอบมุมมองต่อธุรกิจเครื่องดื่มไทย

ระบบไม่จำเป็นต้องติดตามแบรนด์ Sappe / Mogu Mogu โดยตรง เพราะข้อมูลศุลกากรหรือข้อมูลรัฐมักไม่แยกถึงระดับแบรนด์ แต่ให้ใช้หมวดสินค้าตามพิกัดศุลกากรเป็น proxy

## ผู้ใช้/ผู้อ่าน

- นักลงทุนรายบุคคลที่สนใจหุ้นกลุ่มเครื่องดื่ม
- นักวิเคราะห์หุ้นหรือผู้ทำ research ด้านอุตสาหกรรม consumer / beverage
- ผู้ใช้ที่ต้องการดูข้อมูลส่งออกเชิง macro ก่อนนำไปวิเคราะห์บริษัทจดทะเบียนเพิ่มเติม

## ขอบเขต

### ทำ

- ทำ dashboard ติดตามการส่งออกหมวดเครื่องดื่มที่ใกล้เคียงกับ Sappe / Mogu Mogu
- เริ่มจากพิกัดศุลกากรหลัก `HS 2202` ซึ่งเกี่ยวกับน้ำ น้ำแร่ น้ำอัดลมที่เติมน้ำตาล/สารให้ความหวาน หรือปรุงกลิ่นรส และเครื่องดื่มอื่น ๆ ที่ไม่มีแอลกอฮอล์
- แสดงข้อมูลตามเดือน ปี ประเทศปลายทาง มูลค่าส่งออก ปริมาณส่งออก หากข้อมูลมี และการเปลี่ยนแปลง MoM / YoY
- แสดงประเทศปลายทาง Top 10, แนวโน้มรายเดือน, สัดส่วนรายประเทศ และสัญญาณเร่ง/ชะลอของการส่งออก
- แสดงยอดส่งออก Top 10 ของแต่ละประเทศปลายทาง ว่าประเทศนั้นนำเข้าสินค้าหมวดใดจากไทยมากที่สุด พร้อมกราฟเปรียบเทียบ
- มีส่วนแนะนำวิธีดึงข้อมูลอัตโนมัติจากเว็บกรมศุลกากรหรือแหล่งราชการที่เกี่ยวข้อง
- ระบุแหล่งที่มาของข้อมูลทุกชุดในระบบ
- ออกแบบเป็น Web App เขียนโค้ดเอง เพื่อให้ต่อยอดฟีเจอร์ได้ยืดหยุ่น

### ไม่ทำ

- ไม่สรุปว่าเป็นยอดส่งออกของ Sappe / Mogu Mogu โดยตรง หากไม่มีข้อมูลแยกแบรนด์
- ไม่ให้คำแนะนำซื้อ/ขายหุ้นโดยตรง
- ไม่ทำระบบพยากรณ์ขั้นสูงในเวอร์ชันแรก เว้นแต่เพิ่มเป็น phase ถัดไป
- ไม่ scrape ข้อมูลในลักษณะที่ขัดเงื่อนไขการใช้งานของเว็บราชการ

## ข้อมูล/ทรัพยากรที่มี

ผู้ใช้ยังไม่มีข้อมูลในมือ จึงต้องให้ระบบแนะนำและ/หรือเตรียม pipeline สำหรับดึงข้อมูลจากแหล่งราชการ

แหล่งข้อมูลหลักที่ควรพิจารณา:

- กรมศุลกากร: หน้า “รายงานสถิติ” มีตัวเลือก Type, HS-Code, Country, Month, Year และปุ่ม Export CSV  
  URL: https://www.customs.go.th/statistic_report.php
- กรมศุลกากร / Tariff e-Service: ใช้ตรวจสอบพิกัดอัตราศุลกากร  
  URL: https://tariffeservice.customs.go.th/ITRF/index.jsp
- เอกสารสถิติของด่านศุลกากรมุกดาหารพบ `HS 2202` สำหรับหมวดเครื่องดื่มไม่มีแอลกอฮอล์ และระบุแหล่งข้อมูลจาก IMTS / CIS ของกรมศุลกากร  
  URL: https://mukdahan.customs.go.th/data_files/d8b45eb4b7ef6eaffaeb9fc1dca20db5.pdf
- กระทรวงพาณิชย์หรือระบบสถิติการค้าระหว่างประเทศของไทย ควรตรวจสอบเพิ่มเติมในขั้นออกแบบ data pipeline

หมายเหตุด้านพิกัด:

- Candidate HS code เริ่มต้น: `2202`
- ต้องตรวจสอบรหัสละเอียดระดับ 8 หรือ 11 หลักอีกครั้งก่อนใช้งานจริง เพราะกรมศุลกากรระบุว่าเฉพาะ HS 11 หลักมีรายละเอียดปริมาณ
- อาจต้องแยกรายการย่อย เช่น เครื่องดื่มที่เติมน้ำตาล/สารให้ความหวาน เครื่องดื่มปรุงกลิ่นรส หรือเครื่องดื่มไม่มีแอลกอฮอล์ประเภทอื่น

## รูปแบบผลลัพธ์

ผลลัพธ์หลักเป็น Web App แบบ Custom Dashboard ประกอบด้วย:

- หน้า Overview: มูลค่าส่งออกรวม ปริมาณรวม ถ้ามี, MoM, YoY, ประเทศปลายทางหลัก
- หน้า Trend: กราฟรายเดือน/รายปี แยกตามประเทศหรือ HS code ย่อย
- หน้า Country: ตารางและกราฟ Top destination countries
- หน้า Country Detail: เมื่อเลือกประเทศปลายทาง ให้แสดงสินค้าส่งออกจากไทยไปประเทศนั้น Top 10 ตามมูลค่า พร้อมกราฟแท่ง/treemap และแนวโน้มรายเดือนของแต่ละสินค้า
- หน้า Data Source: แสดงแหล่งข้อมูล วันที่ดึงข้อมูลล่าสุด และสถานะการอัปเดต
- หน้า Methodology: อธิบายว่าใช้ HS code ใดเป็น proxy และมีข้อจำกัดอะไร

รูปแบบข้อมูลภายในระบบ:

- ตาราง `exports` สำหรับข้อมูลส่งออก เช่น period, hs_code, country, value_fob, quantity, unit, source, downloaded_at
- ตาราง `hs_codes` สำหรับ mapping พิกัดและคำอธิบายสินค้า
- ตารางหรือ materialized view `country_top_exports` สำหรับคำนวณ Top 10 สินค้าส่งออกของแต่ละประเทศตามช่วงเวลา เช่น period, country, hs_code, product_description, value_fob, rank
- ตาราง `data_runs` สำหรับ log การดึงข้อมูลอัตโนมัติ

## แนวทางดึงข้อมูลอัตโนมัติ

ระบบควรเริ่มจากแนวทางที่เคารพข้อกำหนดของเว็บราชการ:

1. ตรวจสอบว่าหน้าเว็บกรมศุลกากรมี endpoint หรือไฟล์ CSV ที่ export ได้โดยตรงหรือไม่
2. หากมี CSV export ให้สร้าง scheduled job ดึงข้อมูลตามเดือน ปี HS code และประเทศ
3. หากไม่มี API ที่ชัดเจน ให้ใช้ browser automation เฉพาะในกรณีที่ไม่ขัดเงื่อนไขการใช้งาน และควรมี rate limit
4. เก็บ raw CSV/response ไว้แยกจากข้อมูลที่ transform แล้ว เพื่อ audit ย้อนหลังได้
5. ทำ data validation เช่น เช็กเดือนซ้ำ ประเทศซ้ำ หน่วยข้อมูล และช่องว่างของข้อมูล
6. แสดง timestamp และ source URL ใน dashboard ทุกครั้ง

## เกณฑ์วัดความสำเร็จ

- ผู้ใช้สามารถเห็นภาพรวมการส่งออกหมวดเครื่องดื่มที่เกี่ยวข้องได้ภายใน 1-2 นาที
- Dashboard แสดงแนวโน้มมูลค่าส่งออก รายเดือน รายปี MoM และ YoY ได้
- Dashboard แสดงประเทศปลายทาง Top 10 และการเปลี่ยนแปลงของแต่ละประเทศได้
- Dashboard แสดง Top 10 สินค้าส่งออกของไทยไปยังแต่ละประเทศปลายทางได้ พร้อมกราฟที่อ่านง่าย
- ระบบระบุแหล่งข้อมูลและวันที่อัปเดตล่าสุดได้ชัดเจน
- ระบบมีแนวทางหรือ workflow สำหรับดึงข้อมูลซ้ำ/อัปเดตข้อมูลใหม่ได้
- ผู้ใช้เข้าใจข้อจำกัดว่า dashboard เป็นข้อมูลหมวดสินค้า ไม่ใช่ยอดขายหรือยอดส่งออกของแบรนด์ Sappe / Mogu Mogu โดยตรง

## ข้อสมมุติ

- ใช้ `HS 2202` เป็น proxy เริ่มต้นของเครื่องดื่มใกล้เคียง Sappe / Mogu Mogu
- ข้อมูลรัฐไม่แยกถึงระดับแบรนด์ จึงวิเคราะห์ในระดับหมวดสินค้าและประเทศปลายทาง
- แหล่งข้อมูลหลักควรเป็นกรมศุลกากร เพราะเป็นหน่วยงานที่รับผิดชอบข้อมูลนำเข้า-ส่งออกและพิกัดศุลกากรโดยตรง
- ระบบเวอร์ชันแรกเน้น dashboard และ data pipeline เบื้องต้น ยังไม่รวม machine learning หรือ forecasting ขั้นสูง
- ต้องตรวจสอบรหัส HS ระดับละเอียด 8/11 หลักอีกครั้งก่อนเริ่ม implement จริง

## สิ่งที่ต้องยืนยันก่อนลงมือทำ

- จะใช้ `HS 2202` ทั้งหมวด หรือเลือกเฉพาะ HS code ย่อยที่ใกล้กับสินค้า Sappe / Mogu Mogu มากที่สุด
- ต้องการดึงข้อมูลย้อนหลังตั้งแต่ปีใด
- ต้องการอัปเดตข้อมูลถี่แค่ไหน เช่น รายเดือนหลังกรมศุลกากรเผยแพร่ข้อมูลล่าสุด
- ต้องการ tech stack ใดสำหรับ Web App เช่น Next.js, React + Node.js, Python dashboard หรือ stack อื่น
- ต้องการ deploy ที่ไหน เช่น local, Vercel, VPS หรือ internal server
